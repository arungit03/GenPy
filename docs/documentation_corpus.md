# Documentation Corpus Pipeline

The documentation corpus pipeline prepares local educational text for GenPy
continued pretraining. It never downloads files, never contacts GitHub, and does
not modify model checkpoints or training code.

## Input Layout

Create or populate these folders locally:

```text
python_corpus/
  docs/
  peps/
  tutorials/
```

Supported source files:

- Markdown: `.md`
- reStructuredText: `.rst`
- HTML: `.html`
- Plain text: `.txt`

## Run

```bash
python scripts/build_documentation_corpus.py --config configs/documentation_corpus.yaml --force
```

## Cleaning

The processor removes common documentation chrome:

- navigation menus
- table-of-contents blocks
- breadcrumbs
- headers and footers
- copyright blocks
- HTML tags
- JavaScript and CSS
- autogenerated navigation links
- duplicate whitespace

The processor preserves educational content:

- headings
- tutorials and explanations
- API reference text
- Python examples
- fenced and indented code blocks
- tables

## Configuration

```yaml
version: 1
project_root: ..

documentation_corpus:
  corpus_root: python_corpus
  source_directories:
    - docs
    - peps
    - tutorials
  tokenizer: data/tokenizer/tokenizer.json
  min_file_size: 80
  max_file_size: 2000000
  allowed_extensions: [.md, .rst, .html, .txt]
  deduplication: true

  packing:
    context_length: 1024
    max_tokens_per_shard: 10000000
    shard_prefix: documentation
    add_bos: false
    add_eos: true
    document_boundary: eos
    pad_final_sequence: false
```

## Known Limitations

- Deduplication is exact after cleaning and normalization; near-duplicate
  detection is not included.
- HTML cleanup uses deterministic local heuristics, not a browser renderer.
- Some project-specific sidebars may require adding additional ignored markers.
- Very small snippets can be filtered by `min_file_size`; tune the config for
  documentation sets with many short API pages.
