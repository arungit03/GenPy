# Phase 6 GPT Pretraining

Phase 6 is the official GenPy base-model pretraining engine. It trains a
decoder-only GPT model from the packed binary corpus generated by Phase 5.5C.

## Run

Build the final corpus first:

```bash
python scripts/build_pretraining_corpus.py
```

Then start pretraining:

```bash
python scripts/pretrain_gpt.py
```

Useful overrides:

```bash
python scripts/pretrain_gpt.py --max-steps 100 --device cpu --no-resume
python scripts/pretrain_gpt.py --resume-from checkpoints/last_checkpoint.pt
```

## Configuration

Phase 6 uses modular YAML files:

- `configs/model.yaml`: decoder architecture, small/medium/large profiles,
  embedding type, activation, dropout, gradient checkpointing, and
  `torch.compile`.
- `configs/training.yaml`: packed shard paths, batch size, mixed precision, device,
  validation cadence, checkpoint cadence, resume, metrics, samples, and
  TensorBoard output.
- `configs/optimizer.yaml`: AdamW settings.
- `configs/generation.yaml`: sample-generation prompts and decoding settings.

The active model defaults to the Small profile:

- 6 layers
- hidden size 512
- 8 attention heads
- context length 1024

Medium and Large profile values are included in `configs/model.yaml` for
copying into the active `model` section when scaling.

## Precision Support

Phase 6 uses one canonical setting:

```yaml
pretraining:
  training:
    mixed_precision: none
```

Supported values are `none`, `fp16`, and `bf16`.

- `none` uses full fp32 precision and is the default.
- `fp16` enables AMP float16 where the selected device supports it.
- `bf16` enables AMP bfloat16 where the selected device supports it.

For backward compatibility, older YAML files that still contain
`training.precision: fp32` are accepted and automatically mapped to
`mixed_precision: none`.

Device behavior:

- CUDA keeps the existing AMP behavior: `none`, `fp16`, and supported `bf16`
  are valid.
- Apple MPS never defaults to `bf16`. If `bf16` is requested, Phase 6 logs a
  warning and falls back to `none`. If `fp16` is requested, Phase 6 uses it only
  when the installed PyTorch/MPS stack supports MPS autocast; otherwise it logs
  a warning and falls back to `none`.
- CPU should use `none` for Phase 6 pretraining.

## Data

The trainer reads:

- `data/pretraining/index.json`
- `data/pretraining/manifest.json`
- `data/pretraining/shard_*.bin`
- `data/tokenizer/tokenizer.json`

The packed shard loader validates tokenizer hash, vocabulary size, binary
format, dtype, byte order, shard sizes, and sequence length. It supports
memory-mapped reads with deterministic shuffling and an epoch-aware sampler that
is ready for distributed partitioning.

## Checkpoints

Checkpoints are written to `checkpoints/`:

- `step_00001.pt`, `step_00002.pt`, ...
- `last_checkpoint.pt`
- `best_model.pt`

Each checkpoint includes model, optimizer, scheduler, scaler, epoch, global
step, RNG state, model configuration, tokenizer hash, manifest hash, shard index
hash, and training statistics.

## Metrics And Samples

Metrics are written to:

- `metrics/training_metrics.csv`
- `metrics/training_metrics.jsonl`
- `tensorboard/`

Generated code samples are written to:

- `generated_samples/step_XXXXX.json`

Recorded metrics include training loss, validation loss, perplexity, learning
rate, gradient norm, tokens/sec, examples/sec, elapsed time, and GPU memory when
CUDA is available.

## Notes

The current production path uses the existing efficient causal-attention
fallback. Learned, sinusoidal, and rotary positional embeddings are supported.
Deeper distributed strategies are represented as configuration boundaries for
future implementation, without requiring changes to the trainer or data
contracts.
